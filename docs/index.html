HackerHero Documentation

Red Team Operations Manager — local-first, offline-capable, open source.

Usage Guide

HackerHero runs entirely in your browser. No server, no internet connection required. All data is stored in the browser's IndexedDB persistent storage.

Quick Start: Serve the project folder with python3 -m http.server 8080 and open http://localhost:8080.

First Launch

  • On first launch, a random Magic: The Gathering creature name is assigned as your operator name.
  • Change it in Settings → Operator Identity or click your name in the header.
  • Choose a UI theme from the header dropdown or the Settings panel.

Operations

  • Click + New Operation to create a red team operation.
  • Fill in: Code Name (required), Targets (comma-separated: company names, IPs, domains…), Objectives (one per line), and Context notes.
  • Click → Enter on an operation to activate it. The operation name appears in the header badge.
  • Inline rename: click the operation name in the header badge to rename it instantly. Press Enter to save, Escape to cancel. Duplicate codenames are prevented.
  • Click ✎ Edit on an operation card for the full edit form (targets, objectives, context, timezone).
  • Click ⬇ Export to export the operation to a JSON file.
  • Click ⬆ Update to merge data from a JSON file into this operation.
  • Click 🔀 Merge to merge two existing operations (source data merges into target; each operation keeps its own codename).
  • Click 📦 Archive to hide the operation from the main list (restorable).
  • Click ✕ Delete to permanently delete the operation and all its data.

Overview

  • After entering an operation, the Overview panel shows mission info, targets, objectives, context notes, and summary stats.
  • Add or remove targets dynamically (company names, IPs, domains).
  • Manage objectives with a lifecycle: pending → in-progress → achieved → dropped.
  • Edit the free-text context / notes area for background information on the target.
  • Stat cards provide quick links to Assets, Zones, Tickets, and Changelog panels.

Zones & Assets

  • In the Assets panel, the Internet zone is created automatically for every operation.
  • Add custom zones (DMZ, Internal, VPN, Cloud…) with + Zone. Each zone can have a custom color.
  • Select a zone to view and manage its assets. Use All Zones for an aggregated view across every zone.
  • Asset types: Machine 🖥️, Application 📦, Service ⚙️, Credential 🔑, Finding 🐛, Other 📄.
  • Assets support tree nesting — click + on an asset to add a child.
  • Mark an asset as Key Asset ★ to highlight it on the network map.
  • Set a status badge on any asset: Owned 🔓, HVT 🎯, Interesting 👁, Todo 📌, Done ✅.

Data Items (Subitems)

  • Click 📄 on an asset to add a data item (paste command output, credentials, notes).
  • When pasting content, the auto-parser detects common formats and suggests a name. Click Apply to accept.
  • The app recognises 118 command output formats in 26 categories including: nmap, Nuclei, SQLMap, Gobuster, ffuf, httpx, Amass, Subfinder, Katana, Shodan, LinPEAS, Impacket, TruffleHog, AWS CLI, and many more system tools (ip addr, netstat, ps aux, systeminfo, /etc/passwd, /etc/shadow, sudo -l, crontab, SSH keys, HTTP headers, hashes, etc.).
  • Hover over any asset or data item to see the operator who added it and the timestamp.

Version History & Diff

  • Every asset and subitem edit saves a version snapshot with timestamp and operator name.
  • Use the version dropdown to compare any two versions side-by-side.
  • The diff view uses LCS-based line-level comparison with color-coded additions (green) and deletions (red).
  • Click Restore to revert an asset or subitem to any previous version.

Drag & Drop

  • Assets can be dragged and dropped between zones to reassign them.
  • Drop an asset onto a different zone in the sidebar to move it.

Keyboard Navigation

  • ↑ / ↓ — navigate between assets in the tree.
  • Enter / Space — select the highlighted asset.
  • Tab — move focus between the three columns (zones, assets, data).

Column Resize

  • The three-column layout (zones / assets / data viewer) features draggable resize handles.
  • Drag the dividers between columns to adjust widths to your preference.

Search

  • Type any keyword and press Enter or click Search.
  • Results are grouped by type: Zones, Assets, Data Items, Objectives.
  • Use the status filter bar to narrow results by entity type.
  • Click Advanced ▾ to filter by: date range, operator name, zone, asset type.
  • Enable Regex mode to use regular expressions in the query.
  • Matching text is highlighted for easy scanning.

Network Map

  • The Map panel renders all zones and their assets as an interactive SVG diagram.
  • Zones are displayed as colored rectangles with an auto-grid layout of assets inside.
  • Key assets are shown with a gold dot ★.
  • Pan by clicking and dragging. Zoom with the mouse wheel.
  • Click Hide Non-Key to focus on key assets only.
  • Click Fit to reset zoom and pan.
  • Click ⬇ SVG to export the map as a standalone SVG file.

Changelog

  • Every create, update, and delete is automatically recorded with timestamp and operator name.
  • The Changelog panel shows all entries for the active operation, newest first.
  • Use the filter dropdowns to narrow entries by operator, action (create, update, delete, merge), or source operation (for entries imported via merge).
  • Merge entries are highlighted with a 🔀 badge and show the source operation name.
  • Click the revert button ⌫ on any non-merge entry to undo that change.
  • Reverting a create deletes the entity. Reverting an update restores the previous state. Reverting a delete recreates the entity.
  • Click any changelog entry to navigate directly to the referenced entity in the Assets panel.

Tickets

The Tickets panel provides a lightweight issue-tracking system embedded directly in each operation.

  • Create tickets from the Tickets panel or directly from an asset / zone / subitem context menu.
  • Each ticket has a title, description, priority, and status.
  • Priorities: Low 🟢, Medium 🟡, High 🟠, Critical 🔴.
  • Status: Open or Closed. Toggling status is reflected everywhere the ticket is referenced.
  • Tickets can be linked to a zone, asset, or subitem via refType / refId, giving you contextual traceability.
  • Each ticket has a conversation thread — add messages to discuss findings, status updates, or follow-ups.
  • Filter tickets by text search and status (open / closed / all).
  • Linked tickets are also displayed inline in the Assets panel next to their referenced entity.
  • Ticket creation and status changes are recorded in the Changelog.

Image Attachments

Assets, data items (subitems), and tickets support image attachments displayed as an Outlook-style thumbnail strip below each detail view.

  • Add an image: click the + button in the attachment strip. Only image files are accepted (image/*), with a maximum size of 10 MB per file.
  • Where they appear: asset detail view, data item detail view, and ticket detail view — each shows a horizontal strip of 52 px square thumbnails.
  • Lightbox gallery: click any thumbnail to open a full-screen lightbox. Navigate between images with the ← / → arrow keys or the on-screen prev/next buttons.
  • Delete an image: hover over a thumbnail and click the ✕ button to remove it.
  • Thumbnails: auto-generated at 120 px, JPEG quality 0.7, for fast rendering.
  • Storage: images are stored as base64 data URLs in the IndexedDB attachments object store.
  • Cascade delete: when a parent entity (asset, subitem, or ticket) is deleted, all its attachments are automatically removed.
  • Import / Export: attachments are included in JSON exports. The export format version has been bumped to 2 to account for attachment data.
Tip: Keep image sizes reasonable. Large screenshots are resized for thumbnails but the full-resolution original is stored in IndexedDB as base64, which can increase export file size.

Import / Export / Merge

HackerHero supports 3 import workflows for team collaboration:

Export

  • Export any individual operation via ⬇ Export on its card.
  • Export all operations at once via Settings → Export All Operations.
  • Export format includes all data: zones, assets, subitems, versions, tickets, messages, attachments, and changelog.

Import (create new)

  • Click ⬆ Import in the Operations panel to create new operations from a JSON file.
  • If a codename already exists, it is auto-suffixed with -01, -02, etc.
  • All IDs are remapped to prevent collisions. Original operator names and timestamps are preserved.
  • Both single-mission and multi-mission backup formats are supported.

Update (merge from file)

  • Click ⬆ Update on an operation card to merge data from a JSON file into it.
  • Uses smart merge: existing entities are updated if the source is newer (updatedAt-based), new entities are created.
  • Each operation keeps its own codename — the merge never overwrites the destination's name.
  • Merge entries in the changelog are tagged with the source operation name and a 🔀 badge.

Merge two operations

  • Click 🔀 Merge in the Operations panel to merge two existing operations.
  • Select a source and a target — all source data is merged into the target.
  • Conflict detection: when both operators edited the same entity, the merge keeps the newest version and logs a ⚠️ conflict in the changelog.
  • After merge, the source operation remains untouched (you can delete it manually if desired).
Tip: Use the source operation filter in the Changelog panel to review all entries that came from a specific merge.

Settings

  • Operator Identity: change your operator name or click the dice button to get a new random Magic creature name.
  • Theme: pick from 8 visual themes displayed as cards with live swatches.
  • Export All: export every operation as a single JSON backup file.
  • Clear All Data: delete all operations and reset the database. Requires double confirmation.
  • Database Reset (Nuclear): completely destroy and recreate the IndexedDB. Use only as a last resort.

Application Architecture

HackerHero is a local-first single-page application (SPA). It requires only a basic HTTP server to serve static files. No backend, no build step, no npm install.

Technology Stack

LayerTechnologyNotes
UIVanilla HTML + CSS + ES6 JSNo frameworks. Zero dependencies.
StorageIndexedDBVia a Promise-based wrapper in db.js — 12 object stores, schema v7
ModulesES6 native modulestype="module" — 16 JS modules, requires HTTP server
StylingCSS Custom Properties8 themes, zero pre-processing

Data Flow

  • User interaction → Panel method → DB.* → IndexedDB
  • Every mutation also calls logChange() to record a changelog entry.
  • UI re-renders by calling Panel.render() which regenerates the panel's HTML.
  • No virtual DOM, no reactivity framework — simple, explicit DOM manipulation.

Offline & Privacy

  • Zero network requests after initial load. Works in air-gapped environments.
  • All data stays in the browser's IndexedDB on the local machine.
  • Data can be exported as JSON for backup or cross-machine transfer.

Key Relationships

  • Mission → has many Zone, Asset, Ticket, ChangelogEntry
  • Zone → has many Asset (root assets, parentId = null)
  • Asset → has many child Asset (via parentId), Subitem, AssetVersion
  • Subitem → has many SubitemVersion (version history for data items)
  • Ticket → linked to a Zone, Asset, or Subitem via refType / refId; has many TicketMessage
  • ChangelogEntry stores full previousState and newState objects for revert capability

Extending HackerHero

HackerHero is designed for easy extension. Common extension points are described below.

Adding a New Command Output Parser

Open js/parsers.js and follow these steps:

// 1. Create your parser object
const MY_PARSER = {
  id:    'my_parser',      // unique string ID
  label: 'My Tool output', // shown in auto-detect UI
  icon:  '🔧',

  // detect(text) → confidence 0..1
  // Return > 0.1 to appear as a candidate
  detect(text) {
    let s = 0;
    if (/my unique pattern/i.test(text)) s += 0.5;
    if (/another pattern/.test(text))    s += 0.5;
    return Math.min(1, s);
  },

  // parse(text) → structured data object (anything you want)
  parse(text) {
    return { lines: text.split('\\n').filter(Boolean) };
  },

  // Suggested asset name (or null)
  suggestName(parsedData) { return null; },

  // Suggested subitem label
  suggestItemName()       { return 'my tool'; },
};

// 2. Add to the appropriate category section in the PARSERS array
//    (at the bottom of parsers.js — currently 118 parsers in 26 categories)
export const PARSERS = [
  // ... Network / Identity ...
  // ... Scanning ...
  // ... (other categories) ...
  MY_PARSER,   // ← add here or in the relevant category
];

That's it. The auto-detect dialog will pick it up automatically next time the user pastes content into a data item.

Adding a New Theme

Step 1: Add CSS custom properties in css/app.css:

[data-theme="my-theme"] {
  --c-bg:          #…;
  --c-bg2:         #…;
  --c-bg3:         #…;
  --c-border:      #…;
  --c-text:        #…;
  --c-text2:       #…;
  --c-text3:       #…;
  --c-accent:      #…;
  --c-accent-hover:#…;
  --c-accent2:     #…;
  --c-success:     #…;
  --c-warning:     #…;
  --c-danger:      #…;
  --c-info:        #…;
  --c-key-badge:   #…;
  --glow-accent:   …;
  --glow-cyan:     …;
}

Step 2: Register in ThemeManager.themes in js/core.js:

ThemeManager.themes['my-theme'] = {
  name:   'My Theme',
  desc:   'Short description',
  swatch: ['#bg-color', '#accent-color'],
};

Step 3: Add an option in the header select in index.html:

<option value="my-theme">⬛ My Theme</option>

Adding a New Panel

Step 1: Add a panel container in index.html:

<section id="panel-mypanel" class="panel" role="main"></section>

Step 2: Add a nav item in the sidebar:

<li class="nav-item" data-panel="mypanel" title="My Panel">
  <span class="nav-icon">🚀</span>
  <span class="nav-label">My Panel</span>
</li>

Step 3: Create a new module js/panel-mypanel.js:

import { qs } from './core.js';

export const MyPanel = {
  async render() {
    qs('#panel-mypanel').innerHTML = `<div class="panel-header">
      <h2>My Panel</h2>
    </div>
    <p>Hello from MyPanel!</p>`;
    this.bindEvents();
  },
  bindEvents() {
    // Attach DOM event listeners here
  },
};

Step 4: Import it in js/app.js and register in the panel router:

import { MyPanel } from './panel-mypanel.js';
// then in the showPanel() switch:
case 'mypanel': await MyPanel.render(); break;
Tip: If the panel should only be accessible when an operation is active (like Assets or Search), add class="nav-item mission-only" to the nav item. It will be greyed out until an operation is entered.
